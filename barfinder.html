BarFinder (Databases)

March - May, 2020 Onur Sasmaz

Mobile App Development

Uses: SQL, MySQL, HTML, PHP

Goal:

The goal of this project was to design and implementation of relational databases, including data modeling with ER or UML diagrams, relational schema, SQL, relational algebra, user interfaces and administration.

After completion of these projects, I have the ability to:

  • Describe the difference between a relational database and a flat file.
  • Model a moderately complex data set by using an ER diagram, and derive a relational  schema from that diagram.
  • Create a relational database from a relational schema.
  • Create multiple indices in a relational database, and explain when and why such indices  are appropriate.
  • Formulate SQL statements for data manipulation.
  • Formulate simple queries in relational algebra by using projection, selection, product, and  join operations.
  • Describe the components and interfaces of a Web-based database system.
  • Design and implement a Web-based relational database system, using one or more  scripting languages (e.g., PHP) and an open-source database development system (e.g., MySQL).

The Project:

BarFinder is an application that will help you find bars. You can find the the bars address the food they have on their menu, the times they play their different types of music, and more exciting information. If you know a bar, or information about a bar, that is not on BarFinder, you can add it to help other users experience the most out of the application. Last but not least, the user can look at all the tables the different bars provide, to find the perfect table. To gain access to all this you just have to sign up.
