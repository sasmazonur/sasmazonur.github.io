Parallel Programming

March - May, 2020 Onur Sasmaz

Uses: C++, CUDA, OpenCL, OpenMP

The goal of these projects were:

These applications designed to improve user interaction and simulation and computational performance. The goals of these projects are to improve tasks that require desktop parallelism, both on a CPU and a GPU.

During these projects I had the change to work with 16 NVidia Tesla V100 GPUs. That Each card has 5,120 CUDA Cores and 640 Tensor Cores. This gives each 16 V100 DGX server a total of 81,920 CUDA cores and 10,240 Tensor cores.

These projects include:

  1. Parallel computing: types, limitations
  2. Moore’s Law, Amdahl’s Law
  3. OpenMP
  4. Synchronization issues in parallel computing
  5. Cache issues in parallel computing
  6. SIMD
  7. GPU computing
  8. OpenCL
  9. CUDA

After completion of these projects, I have the ability to:

  • Explain the clockspeed limitations of computing using physics and Moore's Law
  • Explain the limitations of parallel computing using Amdahl's Law
  • Demonstrate "parallel thinking" in program design
  • Demonstrate the ability to program parallel algorithms in multicore, SIMD, and GPU.
  • Characterize what types of problems are best able to be parallelized
  • Characterize how cache issues affect parallel performance
  • Demonstrate the proper use of synchronization to avoid race conditions and deadlock
  • Characterize the benefits of using a CPU versus using a GPU for parallel programming
  • Characterize the benefits of using a GPU versus using a CPU for parallel programming
